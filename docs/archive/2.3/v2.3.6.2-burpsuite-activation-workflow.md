## Container Behavior - rtpi-burp-agent

### Dormant Mode Entrypoint

**File:** `docker/offsec-agents/burp-entrypoint.sh`

**Behavior:**
1. Container starts in dormant mode (monitoring loop)
2. Waits for JAR + license files in `/opt/burp-setup/`
3. Waits for activation flag: `/opt/burp-setup/.activate`
4. On flag detected: Starts Burp Pro + MCP Server
5. On flag removed: Stops Burp and returns to dormant mode

See full bash script in document body above.

## Agent State Machine

```typescript
// State progression:
dormant → activating → active → idle → running → idle
                    ↓           ↓
                  error      error
                    ↓           ↓
                dormant     dormant
```

**Task Rejection When Dormant:**
- Agent checks activation status before executing tasks
- If dormant: Reports blocker to Operations Manager
- Returns error with `requiresActivation: true`

## Security Considerations

1. **File Validation** - Magic bytes, size limits, format checks
2. **Access Control** - Admin-only uploads, audit logging
3. **Container Isolation** - Volume-based file access only
4. **Error Handling** - Automatic rollback, clear error messages
