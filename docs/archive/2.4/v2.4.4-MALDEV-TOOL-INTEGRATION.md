## Overview

This enhancement integrates the 44 reverse-engineering, ROP development, and evasion tools available in the `rtpi-maldev-agent` container with the maldev-agent workflow, transforming it from AI-only code generation to tool-assisted exploit development.

## Problem Statement

The maldev-agent container has 44 powerful security tools installed but the agent never uses them:
- Reverse Engineering: Ghidra MCP, radare2, x64dbg, WinDbg
- ROP Development: ROPgadget, RustChain, ropium, xgadget  
- Shellcode/Evasion: pwntools, TamperETW, SHAPESHIFTER, SysWhispers
- Validation: unicorn, capstone, keystone

The agent only uses AI (Claude/GPT/Ollama) to generate exploit code, missing opportunities for tool-assisted analysis and development.

## Architecture

### Tool Execution Flow

```
Maldev Agent
    ↓
  useToolIntegration=true?
    ↓ Yes
  maldevToolExecutor.generateShellcode()
    ↓
  DockerExecutor.exec(rtpi-maldev-agent, python3 script.py)
    ↓
  Parse shellcode output
    ↓
  Pass to AI in enhanced prompt
    ↓
  Generate MSF module with tool insights
```

### Key Design Decisions

1. **Opt-In Tool Integration** - Maintains backward compatibility
2. **Container User Separation** - Each container has its own user (rtpi-agent, rtpi-tools)
3. **Script-Based Execution** - Generate Python scripts for complex tools (pwntools)
4. **Direct CLI for Simple Tools** - Execute radare2, ROPgadget directly
5. **Result Parsing** - Extract structured data from tool output

## Known Limitations

### 1. ARM64/x86_64 Cross-Compilation

**Issue:** ARM64 container cannot assemble x86_64 shellcode

**Impact:** pwntools `asm()` function fails on ARM64 hosts

**Workaround:**
- Use shellcraft templates without assembly
- Use pre-compiled Metasploit payloads
- Focus on ROP chain logic generation

**Long-term Fix:**
```dockerfile
# Add to Dockerfile.maldev-tools
RUN apt-get update && apt-get install -y \
    binutils-x86-64-linux-gnu \
    binutils-i686-linux-gnu \
    && rm -rf /var/lib/apt/lists/*
```

### 2. Binary File Requirement

**Issue:** Many tools require actual binary files to analyze

**Impact:** Cannot test radare2, ROPgadget without target binaries

**Solution:** 
- Tools work when PoC provides binary
- For web vulnerabilities, tool integration is limited
- Focus: Binary exploitation scenarios

### 3. Ghidra MCP Integration

**Status:** Not yet implemented

**Reason:** Ghidra requires GUI or headless mode setup

**Future:** Integrate Ghidra MCP when MCP server available

## Technical Details

### Container Specification

**Container:** `rtpi-maldev-agent`  
**User:** `rtpi-agent`  
**Tools Directory:** `/opt/tools/`  
**Temp Directory:** `/tmp/`

**Installed Tools:**
- Binary Analysis: radare2, binutils, binwalk, pev, upx
- Python Libraries: pwntools, capstone, keystone, unicorn, ropper, angr, pefile, lief
- ROP Tools: ROPgadget, xgadget (Rust)
- Debuggers: gdb, gdb-multiarch, strace, ltrace
- Compilers: gcc, g++, nasm, yasm, mingw-w64 (x86_64 only on AMD64)

### Script Generation Pattern

All complex tool executions follow this pattern:

```typescript
// 1. Generate Python/bash script with tool logic
const script = this.generateScript(parameters);

// 2. Write script to container /tmp
const writeCmd = `cat > ${scriptPath} << 'EOF'\n${script}\nEOF`;
await dockerExecutor.exec(container, ['bash', '-c', writeCmd]);

// 3. Execute script
const execCmd = `cd /tmp && python3 ${scriptPath}`;
const result = await dockerExecutor.exec(container, ['bash', '-c', execCmd]);

// 4. Parse structured output
return this.parseOutput(result.stdout);
```

**Benefits:**
- Complex tool orchestration
- Clean parameter passing
- Structured output parsing
- Error isolation

## ARM64 Cross-Compilation Solution

### Current State

The maldev container runs on ARM64 architecture:
```bash
$ docker exec rtpi-maldev-agent dpkg --print-architecture
arm64
```

Installed binutils:
```
binutils                    # ARM64 binutils
binutils-aarch64-linux-gnu  # ARM64 cross-compiler
```

**Missing:** x86_64 binutils for assembling x86_64 shellcode

### Solution

Add to `docker/offsec-agents/Dockerfile.maldev-tools`:

```dockerfile
# Install x86_64 cross-compilation tools (after existing binutils installation)
RUN apt-get update && apt-get install -y \
    binutils-x86-64-linux-gnu \
    binutils-i686-linux-gnu \
    && rm -rf /var/lib/apt/lists/*
```

This enables pwntools to assemble x86/x64 shellcode on ARM64 hosts.
