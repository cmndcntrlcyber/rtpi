## 1. Current RTPI Architecture Assessment

RTPI orchestrates a multi-container Docker Compose stack aligned to MITRE ATT&CK:

**Existing containers**: evilgophish-rtpi (phishing simulation), Kali AttackBox (Nmap/Metasploit/Hashcat), Windows 7 TargetBox, Ollama (local LLM inference), Open-WebUI (chat interface).

**Agent system**: Dual-track — Python/LangChain `AgentExecutor` with six tools (nmap_tool, metasploit_search_tool, metasploit_process_tool, security_analyst_tool, search_tool, wiki_tool, save_tool), plus a Deno/TypeScript implementation with three-tier hierarchy (PentestOrchestratorAgent → MitrePlannerAgent → ExploitExecutorAgent).

**Multi-model support**: OpenAI GPT-4o mini, Anthropic Claude, Ollama local models (qwen2.5-coder, deepseek-r1), Cloudflare AI Workers.

**What the platform needs**: persistent cross-session memory, a deep skills library beyond the six core tools, workflow branching and conditional logic, reverse engineering capabilities, security architecture review tools, and frontend/backend development skills for building custom tooling.

## 4. mem0 Memory Architecture for Unified Operations

### Four Memory Scopes

**Engagement-level** (`user_id` = target org): Every discovered host, service, vulnerability, credential, and network relationship accumulates here across all sessions. Mem0's conflict resolution reconciles updated findings automatically.

**Agent-specialized** (`agent_id` = agent role): The recon agent, exploitation agent, RE agent, and reporting agent maintain separate knowledge bases while sharing engagement context. Cross-agent queries enable the exploitation agent to retrieve reconnaissance findings without coupling.

**Session memory** (`run_id` = session ID): Individual operation phases — a specific scanning run, exploitation attempt, or RE session. Provides granular audit trails.

**Graph memory** (Neo4j/Apache AGE): Automatically builds a knowledge graph of the engagement. Entities: hosts, services, CVEs, credentials, attack paths, MITRE techniques. Enables queries like "Show all paths from the compromised web server to the domain controller."

## 5. Workflow Orchestration: LangGraph (Not Manaflow)

### Why Manaflow Is Set Aside

Manaflow is specifically a developer tool for managing parallel AI coding agents (Claude Code, Codex CLI, Cursor) in isolated VS Code workspaces. While the sandboxed Docker execution concept has merit, Manaflow provides no workflow DAG/pipeline support, no conditional branching, no persistent memory, no REST API, and runs only on macOS. The engineering effort to repurpose Manaflow would exceed building the orchestration natively.

### Transferable Concepts from Manaflow

The one valuable pattern: **sandbox isolation**. Each security tool execution should run in an isolated container to prevent cross-contamination. RTPI's Docker architecture already supports this natively — extend with per-tool ephemeral containers for payload testing and exploit verification.

### LangGraph Orchestration Design

LangGraph provides stateful, cyclical workflows with conditional branching and human-in-the-loop checkpoints. This maps directly to cybersecurity operations:

**Sequential chains**: Reconnaissance → Vulnerability Assessment → Exploitation → Post-Exploitation → Reporting

**Conditional routing**: Critical vulnerability found? → Route to exploitation. Informational only? → Log and continue scanning. RE needed? → Route binary to Reverse Engineering pipeline.

**Parallel fan-out**: Simultaneously scan multiple hosts, test multiple vulnerability categories, or run multiple RE tools against a sample.

**Human-in-the-loop**: Pause before exploitation for operator approval. Pause before data exfiltration for scope verification. Pause before social engineering execution.

**ClaudeAdvancedPlugins integration point**: Each plugin's system prompt becomes the instruction set for a specialized LangGraph node. The `/pentest-ad` plugin's methodology drives the AD assessment node. The `/reverse-malware` plugin drives the malware RE node. The `/exploit-dev` plugin drives the exploit development node.

## 6. ClawGod Architectural Patterns Worth Adopting

ClawGod reveals multi-agent capabilities through runtime patching of Claude Code. The patterns to adopt natively in RTPI (without binary patching):

**Agent Teams (Multi-Agent Swarms)**: Multiple specialized agents collaborate simultaneously — a recon agent discovers services while an OSINT agent profiles employees while a cloud assessment agent tests IAM. LangGraph's fan-out pattern implements this.

**Ultraplan (Multi-Agent Planning)**: Before execution, multiple agents contribute to a consolidated operation plan. The recon agent proposes scanning strategy, the exploitation agent identifies likely attack paths, the RE agent flags binaries worth analyzing. LangGraph's planning phase node aggregates these proposals.

**Computer Use (GUI Automation)**: For tools with no CLI — GUI-based testing of thick clients, Burp Suite automation, Ghidra scripted analysis. Implement via headless browser automation (Playwright) within tool containers.

### How ClaudeAdvancedPlugins Integrate at Each Layer

**Layer 1 (Execution)**: Plugin system prompts reference specific tools. When `/reverse-firmware` calls for binwalk extraction, the orchestrator routes to a firmware-analysis container pre-loaded with binwalk, QEMU, and Firmadyne. When `/pentest-ad` needs BloodHound, the AD-assessment container has SharpHound and neo4j ready.

**Layer 2 (Skills)**: Each ClaudeAdvancedPlugins markdown file is registered in the skills library alongside Anthropic-Skills. The skill discovery service indexes plugin commands by domain, prerequisite tools, and ATT&CK technique coverage. A query for "Active Directory lateral movement" returns both the Anthropic-Skills AD ACL analysis skill AND the `/pentest-ad` plugin's comprehensive methodology.

**Layer 3 (Orchestration)**: Plugin system prompts become LangGraph node instructions. The `/red-team-ops` prompt defines the behavior of the Red Team Orchestrator node. The `/exploit-dev` prompt defines the Exploitation node. Conditional routing uses mem0 context: if RE analysis reveals a custom protocol, route to the `/reverse-protocol` node.

**Layer 4 (Memory)**: Every plugin execution writes structured findings to mem0. The `/pentest-ad` node stores discovered AD objects, trust relationships, and attack paths in graph memory. The `/reverse-malware` node stores unpacked samples, extracted C2 configs, and IOCs. Cross-plugin queries: the exploitation agent asks "What credentials did the AD assessment discover?" via mem0.search().

**Layer 5 (Interface)**: Plugin slash commands activate via the Open-WebUI chat interface. The custom dashboard (built using Frontend plugins) visualizes the engagement graph, attack paths, and coverage metrics. The `/pentest-report` plugin generates final deliverables from aggregated mem0 data.

## 10. Key Architectural Decisions

1. **Plugin format unification**: ClaudeAdvancedPlugins (Markdown system prompts) and Anthropic-Cybersecurity-Skills (agentskills.io YAML+MD) coexist. The skill discovery service indexes both formats — YAML frontmatter for Anthropic skills, filename/directory parsing for plugins.

2. **Manaflow replaced by LangGraph**: Manaflow's coding-agent management scope does not match RTPI's security workflow needs. LangGraph provides native mem0 integration, conditional branching, and multi-agent coordination. The sandbox isolation pattern from Manaflow is implemented through Docker's existing container isolation.

3. **ClawGod patterns without patches**: Multi-agent swarms, planning phases, and GUI automation are implemented natively through LangGraph fan-out, aggregation nodes, and Playwright-based headless automation.

4. **mem0 graph memory is the differentiator**: The engagement knowledge graph — connecting hosts, services, vulnerabilities, credentials, attack paths, RE findings, and MITRE techniques — transforms RTPI from isolated tool execution into continuous, contextual intelligence.

5. **ClaudeAdvancedPlugins' Context & Memory plugins augment mem0**: The Hallucination Guard's confidence scoring becomes metadata on all mem0 entries. The Context Manager's priority loading informs the skill discovery service. The Token Tracker provides cost awareness for multi-model operations.
