# v2.5.3 — BOF/COFF Loader

## Overview

Implement the BOF (Beacon Object File) loader — the first and most critical loader. BOFs are COFF-format object files used by Cobalt Strike that import "Beacon API" functions for I/O. This phase delivers: COFF parsing with goblin, Unicorn CPU emulator setup, Beacon API emulation hooks, and end-to-end BOF execution with input/output via the staging broker.

## Prerequisites

- Phase 1 complete (bof.rs scaffold, emulation/beacon_api.rs scaffold)
- Phase 2 complete (staging broker handles StageOutput/GetInput)

## Implementation Details

### 3.1 COFF Parsing (`bof.rs`)

Use `goblin::pe::Coff` to parse BOF files:

```rust
pub fn parse_coff(&self, data: &[u8]) -> SandboxResult<ParsedCoff> {
    let coff = goblin::pe::Coff::parse(data)
        .map_err(|e| SandboxError::InvalidFormat(e.to_string()))?;

    let sections = coff.sections.iter().map(|s| {
        let raw_data = &data[s.pointer_to_raw_data as usize
            ..(s.pointer_to_raw_data + s.size_of_raw_data) as usize];
        CoffSection {
            name: s.name().unwrap_or("").to_string(),
            data: raw_data.to_vec(),
            characteristics: s.characteristics,
            virtual_address: s.virtual_address as u64,
        }
    }).collect();

    // Extract symbols — find "go" entry point
    let symbols = coff.symbols.iter().filter_map(|(_, _, sym)| {
        sym.name(&coff.strings).ok().map(|name| CoffSymbol {
            name: name.to_string(),
            value: sym.value,
            section_number: sym.section_number,
            storage_class: sym.storage_class,
        })
    }).collect();

    // ...

    Ok(ParsedCoff { sections, symbols, relocations, entry_symbol: Some("go".into()) })
}
```

### 3.2 Unicorn Emulator Setup

```rust
pub fn create_emulator(&self) -> SandboxResult<Unicorn<EmulatorContext>> {
    let mut emu = Unicorn::new_with_data(
        Arch::X86, Mode::MODE_64, EmulatorContext::new()
    ).map_err(|e| SandboxError::EmulatorInit(e.to_string()))?;

    // Map memory regions per architecture doc
    emu.mem_map(CODE_BASE, CODE_SIZE, Permission::ALL)?;       // 0x400000, 2MB
    emu.mem_map(DATA_BASE, DATA_SIZE, Permission::READ | Permission::WRITE)?;
    emu.mem_map(HEAP_BASE, HEAP_SIZE, Permission::READ | Permission::WRITE)?;
    emu.mem_map(STACK_BASE, STACK_SIZE, Permission::READ | Permission::WRITE)?;

    // Set stack pointer to top of stack
    emu.reg_write(RegisterX86::RSP, STACK_BASE + STACK_SIZE as u64 - 8)?;

    Ok(emu)
}
```

### 3.3 Section Loading

```rust
pub fn load_sections(&self, emu: &mut Unicorn<EmulatorContext>, parsed: &ParsedCoff) {
    for section in &parsed.sections {
        let addr = CODE_BASE + section.virtual_address;
        emu.mem_write(addr, &section.data)?;
    }
}
```

### 3.4 Relocation Processing

Apply COFF relocations (IMAGE_REL_AMD64_*):
- `IMAGE_REL_AMD64_ADDR64` — 64-bit absolute address
- `IMAGE_REL_AMD64_ADDR32NB` — 32-bit relative address
- `IMAGE_REL_AMD64_REL32` — 32-bit PC-relative

### 3.5 Beacon API Emulation (`beacon_api.rs`)

Each Beacon API function is a hook at a known address. When the BOF calls an imported function, the emulator hits the hook address and routes to the Rust handler:

```rust
const BEACON_API_BASE: u64 = 0x00700000; // Dedicated region for API hooks

pub struct BeaconApiTable {
    pub beacon_printf: u64,    // BEACON_API_BASE + 0x00
    pub beacon_output: u64,    // BEACON_API_BASE + 0x10
    pub beacon_data_parse: u64,// BEACON_API_BASE + 0x20
    pub beacon_data_int: u64,  // BEACON_API_BASE + 0x30
    // ... etc
}

// Hook handler: reads format string + args from emulator memory,
// formats output, stages via broker
fn handle_beacon_printf(emu: &mut Unicorn<EmulatorContext>) {
    let format_ptr = emu.reg_read(RegisterX86::RCX).unwrap();
    let format_str = read_string_from_emu(emu, format_ptr);
    let output = format_beacon_string(&format_str, emu);
    emu.get_data_mut().stdout.extend(output.as_bytes());
}
```

**Full Beacon API table (9 functions):**

| Function | Hook Address | Behavior |
|----------|-------------|----------|
| `BeaconPrintf` | BASE+0x00 | Format string → stdout → broker |
| `BeaconOutput` | BASE+0x10 | Raw bytes → stdout → broker |
| `BeaconDataParse` | BASE+0x20 | Parse argument buffer, set data pointer |
| `BeaconDataInt` | BASE+0x30 | Extract i32 from argument buffer |
| `BeaconDataShort` | BASE+0x40 | Extract i16 from argument buffer |
| `BeaconDataLength` | BASE+0x50 | Return remaining data length |
| `BeaconDataExtract` | BASE+0x60 | Extract byte slice from arguments |
| `BeaconFormatAlloc` | BASE+0x70 | Allocate format buffer on heap |
| `BeaconFormatPrintf` | BASE+0x80 | Printf to format buffer |

### 3.6 Import Resolution

When COFF imports reference `__imp_BeaconPrintf` etc., resolve to hook addresses:

```rust
fn resolve_import(&self, name: &str) -> Option<u64> {
    match name {
        "__imp_BeaconPrintf" | "BeaconPrintf" => Some(self.api_table.beacon_printf),
        "__imp_BeaconOutput" | "BeaconOutput" => Some(self.api_table.beacon_output),
        // ...
        _ => None, // Unknown import
    }
}
```

### 3.7 Execution Entry

BOFs use `go(char* args, int args_len)` as entry point:

```rust
pub fn execute_bof(&self, emu: &mut Unicorn<EmulatorContext>,
                    entry: u64, args: &[u8]) -> SandboxResult<()> {
    // Push args onto stack (Windows x64 calling convention)
    // RCX = args pointer, RDX = args length
    let args_addr = HEAP_BASE;
    emu.mem_write(args_addr, args)?;
    emu.reg_write(RegisterX86::RCX, args_addr)?;
    emu.reg_write(RegisterX86::RDX, args.len() as u64)?;

    // Execute from entry point
    emu.emu_start(entry, 0, 0, 0)?;

    Ok(())
}
```
