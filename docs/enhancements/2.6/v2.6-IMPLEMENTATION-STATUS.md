## Executive Summary

The v2.6 enhancement adds `nexus-sandbox` to the `rust-nexus` C2 framework — an embedded userland execution environment that sandboxes binary payloads (BOF/COFF, PE, ELF, WASM, shellcode) via CPU emulation (Unicorn Engine) with capability-based access control and a staging broker for secure bidirectional C2 communication.

## Phase 9: Codesigning with Sigstore (Final Phase)

### Overview

All release artifacts (compiled Rust binaries, container images, proto files) will be cryptographically signed using [Sigstore](https://docs.sigstore.dev/) keyless signing. This provides supply chain integrity verification without managing long-lived signing keys.

### Technology

- **Sigstore Cosign** — Signs container images and binary blobs
- **Keyless signing** — Uses GitHub Actions OIDC tokens (no private keys to manage)
- **Transparency log** — All signatures recorded in Rekor for public verifiability
- **Verification** — Anyone can verify artifacts using the certificate identity + OIDC issuer

### Implementation Plan

#### 9.1 Sign Compiled Binaries

Sign all `cargo build --release` output binaries for each target:

```yaml
# .github/workflows/release.yml
name: Build, Sign, and Release
on:
  push:
    tags: ['v*']

jobs:
  build-and-sign:
    runs-on: ubuntu-latest
    permissions:
      contents: write
      id-token: write    # Required for Sigstore OIDC keyless signing
      packages: write

    steps:
      - uses: actions/checkout@v4
        with:
          persist-credentials: false

      - name: Install Cosign
        uses: sigstore/cosign-installer@v4.0.0

      - name: Install Rust toolchain
        uses: dtolnay/rust-toolchain@stable
        with:
          targets: x86_64-unknown-linux-gnu,x86_64-pc-windows-gnu

      - name: Build release binaries
        run: |
          cargo build --release -p nexus-agent --target x86_64-unknown-linux-gnu
          cargo build --release -p nexus-sandbox --target x86_64-unknown-linux-gnu

      - name: Sign binaries with Sigstore (keyless)
        run: |
          for binary in target/x86_64-unknown-linux-gnu/release/nexus-agent \
                        target/x86_64-unknown-linux-gnu/release/nexus-sandbox; do
            if [ -f "$binary" ]; then
              cosign sign-blob "$binary" \
                --bundle "${binary}.sigstore.json" \
                --yes
              echo "Signed: $binary"
            fi
          done

      - name: Verify signatures
        run: |
          for binary in target/x86_64-unknown-linux-gnu/release/nexus-agent \
                        target/x86_64-unknown-linux-gnu/release/nexus-sandbox; do
            if [ -f "${binary}.sigstore.json" ]; then
              cosign verify-blob "$binary" \
                --bundle "${binary}.sigstore.json" \
                --certificate-identity="https://github.com/${{ github.repository }}/.github/workflows/release.yml@${{ github.ref }}" \
                --certificate-oidc-issuer="https://token.actions.githubusercontent.com"
              echo "Verified: $binary"
            fi
          done

      - name: Upload signed artifacts to release
        uses: softprops/action-gh-release@v1
        with:
          files: |
            target/x86_64-unknown-linux-gnu/release/nexus-agent
            target/x86_64-unknown-linux-gnu/release/nexus-agent.sigstore.json
            target/x86_64-unknown-linux-gnu/release/nexus-sandbox
            target/x86_64-unknown-linux-gnu/release/nexus-sandbox.sigstore.json
```

#### 9.2 Sign Container Images

Sign Docker images pushed to GHCR:

```yaml
      - name: Build and push container image
        uses: docker/build-push-action@v6.18.0
        id: build-and-push
        with:
          push: true
          tags: ghcr.io/${{ github.repository }}/nexus-agent:${{ github.ref_name }}

      - name: Sign container image with Sigstore
        env:
          DIGEST: ${{ steps.build-and-push.outputs.digest }}
        run: |
          cosign sign --yes \
            "ghcr.io/${{ github.repository }}/nexus-agent:${{ github.ref_name }}@${DIGEST}"

      - name: Verify container signature
        env:
          DIGEST: ${{ steps.build-and-push.outputs.digest }}
        run: |
          cosign verify \
            "ghcr.io/${{ github.repository }}/nexus-agent:${{ github.ref_name }}@${DIGEST}" \
            --certificate-identity="https://github.com/${{ github.repository }}/.github/workflows/release.yml@${{ github.ref }}" \
            --certificate-oidc-issuer="https://token.actions.githubusercontent.com"
```

#### 9.3 Sign Proto/Schema Artifacts

Sign the `.proto` files as release artifacts for downstream consumers:

cosign sign-blob nexus-infra/proto/nexus.proto \
  --bundle nexus.proto.sigstore.json --yes

cosign sign-blob nexus-infra/proto/sandbox.proto \
  --bundle sandbox.proto.sigstore.json --yes

#### 9.4 Verification Script for End Users

Provide a verification script users can run after downloading:

```bash
#!/bin/bash
# verify-release.sh — Verify Sigstore signatures on nexus release artifacts
REPO="cmndcntrlcyber/rust-nexus"
ISSUER="https://token.actions.githubusercontent.com"

for artifact in nexus-agent nexus-sandbox; do
  if [ -f "${artifact}" ] && [ -f "${artifact}.sigstore.json" ]; then
    echo "Verifying ${artifact}..."
    cosign verify-blob "${artifact}" \
      --bundle "${artifact}.sigstore.json" \
      --certificate-identity-regexp="https://github.com/${REPO}/.*" \
      --certificate-oidc-issuer="${ISSUER}"
    echo ""
  fi
done
```
