# v2.6.2 — Staging Broker Implementation

## Overview

The Staging Broker mediates **all** communication between sandboxed code and external resources (host filesystem, network, C2 channel). It enforces capability-based access control, tracks resource usage, and maintains an audit log. This is the security backbone of the sandbox — no data enters or leaves without broker validation.

## Implementation Details

### 2.1 Complete BrokerRequest Handlers

The scaffold has `StageOutput` and `GetInput` partially implemented. Complete:

```rust
// ReadHostFile — capability-gated filesystem read
BrokerRequest::ReadHostFile { path, response } => {
    if !area.capabilities.can_read_path(&path) {
        audit_log.push(AuditEntry::denied("read_file", &path));
        return Err(SandboxError::CapabilityDenied(...));
    }
    let data = tokio::fs::read(&path).await?;
    let _ = response.send(Ok(data));
}

// WriteHostFile — capability-gated filesystem write
BrokerRequest::WriteHostFile { path, data, response } => {
    // Sanitize path (no traversal: ../, symlink resolution)
    let canonical = sanitize_path(&path)?;
    if !area.capabilities.can_write_path(&canonical) { ... }
    tokio::fs::write(&canonical, &data).await?;
}

// HostCommand — capability-gated command execution
BrokerRequest::HostCommand { command, args, response } => {
    if !area.capabilities.can_execute_command(&command) { ... }
    let output = tokio::process::Command::new(&command)
        .args(&args)
        .output().await?;
    let _ = response.send(Ok(output.stdout));
}

// AllocateMemory — tracked against max_memory_bytes
BrokerRequest::AllocateMemory { size, response } => {
    let mut used = area.total_memory_used.write();
    if *used + size > area.capabilities.max_memory_bytes {
        return Err(SandboxError::MemoryExhausted);
    }
    *used += size;
    // Return address in emulator heap space
}
```

### 2.2 Audit Logging

```rust
struct AuditEntry {
    timestamp: chrono::DateTime<chrono::Utc>,
    context_id: String,
    operation: String,
    target: String,
    result: AuditResult, // Allowed, Denied, Error
    bytes_transferred: usize,
}
```

Log every request. Expose `get_audit_log(context_id)` for post-execution review.

### 2.3 Path Sanitization

Prevent path traversal attacks:
- Resolve `../` sequences
- Reject absolute paths outside allowed prefixes
- Follow symlinks and re-check against capabilities
- Reject paths containing null bytes

### 2.4 Resource Tracking

Track per-context:
- Total output bytes (enforce `max_output_size_bytes`)
- Total memory allocated (enforce `max_memory_bytes`)
- CPU time consumed (tracked by emulator, checked by broker)
- Number of host requests (for rate limiting)
