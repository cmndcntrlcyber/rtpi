# v2.6.5 — WASM Engine Integration

## Overview

Integrate Wasmtime as a high-performance sandbox for WebAssembly modules. WASM provides ~40% overhead (vs 10-100x for CPU emulation), making it ideal for performance-sensitive payloads. This phase adds WASI filesystem support, custom host functions for broker communication, and fuel-based CPU time limiting.

## Prerequisites

- Phase 2 complete (staging broker for I/O routing)

### 5.1 Wasmtime Engine Setup

```rust
use wasmtime::*;
use wasmtime_wasi::{WasiCtx, WasiCtxBuilder};

pub struct WasmEngine {
    engine: Engine,
}

pub struct WasmState {
    wasi: WasiCtx,
    broker_tx: mpsc::Sender<BrokerRequest>,
    capabilities: Capabilities,
    output_buffer: Vec<u8>,
}

impl WasmEngine {
    pub fn new() -> SandboxResult<Self> {
        let mut config = Config::new();
        config.consume_fuel(true);           // Enable fuel-based CPU limiting
        config.epoch_interruption(true);     // Enable epoch-based interruption

        let engine = Engine::new(&config)
            .map_err(|e| SandboxError::EmulatorInit(e.to_string()))?;

        Ok(Self { engine })
    }
}
```

### 5.2 WASI Configuration

```rust
fn create_wasi_ctx(capabilities: &Capabilities) -> WasiCtxBuilder {
    let mut builder = WasiCtxBuilder::new();

    // Inherit stdout/stderr for output capture
    builder.stdout(Box::new(OutputCapture::new()));
    builder.stderr(Box::new(OutputCapture::new()));

    // Map filesystem paths based on capabilities
    for read_path in &capabilities.fs_read_paths {
        builder.preopened_dir(read_path, read_path, DirPerms::READ, FilePerms::READ);
    }
    for write_path in &capabilities.fs_write_paths {
        builder.preopened_dir(write_path, write_path, DirPerms::all(), FilePerms::all());
    }

    // Environment variables (filtered)
    builder.env("SANDBOX", "1");

    builder
}
```

### 5.4 Fuel-Based CPU Limiting

```rust
pub fn execute_wasm(&self, module_bytes: &[u8], state: WasmState) -> SandboxResult<ExecutionResult> {
    let module = Module::new(&self.engine, module_bytes)?;

    let mut store = Store::new(&self.engine, state);

    // Set fuel limit based on max_cpu_time_ms (rough: 1ms ≈ 10,000 fuel units)
    let fuel = store.data().capabilities.max_cpu_time_ms * 10_000;
    store.set_fuel(fuel)?;

    let mut linker = Linker::new(&self.engine);
    wasmtime_wasi::add_to_linker(&mut linker, |s: &mut WasmState| &mut s.wasi)?;
    register_sandbox_imports(&mut linker)?;

    let instance = linker.instantiate(&mut store, &module)?;

    // Call _start (WASI entry point) or custom entry
    let start = instance.get_typed_func::<(), ()>(&mut store, "_start")?;

    match start.call(&mut store, ()) {
        Ok(()) => { /* success */ }
        Err(e) if e.downcast_ref::<wasmtime::Trap>() == Some(&Trap::OutOfFuel) => {
            return Err(SandboxError::CpuTimeExhausted);
        }
        Err(e) => return Err(SandboxError::ExecutionFailed(e.to_string())),
    }

    // Collect output
    Ok(ExecutionResult::success(store.data().output_buffer.clone(), duration))
}
```

### 5.5 Output Capture

Implement `wasmtime_wasi::StdoutStream` to capture stdout:

```rust
struct OutputCapture {
    buffer: Arc<Mutex<Vec<u8>>>,
}

impl Write for OutputCapture {
    fn write(&mut self, buf: &[u8]) -> io::Result<usize> {
        self.buffer.lock().unwrap().extend_from_slice(buf);
        Ok(buf.len())
    }
    fn flush(&mut self) -> io::Result<()> { Ok(()) }
}
```
