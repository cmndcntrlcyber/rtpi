## v2.7.2 - C2 Framework Docker Deployments **[NOT STARTED]**
**File:** [v2.7.2-c2-framework-deployments.md](v2.7.2-c2-framework-deployments.md)

Deploy 4 new C2 frameworks as independent Docker containers following the dormant activation pattern (like BurpSuite). Each framework deploys one instance per operation, one at a time.

### Frameworks
| Framework | Source | Dockerfile |
|-----------|--------|------------|
| C3 | [ReversecLabs/C3](https://github.com/ReversecLabs/C3) | `docker/offsec-agents/Dockerfile.c3-tools` |
| Sliver | [BishopFox/sliver](https://github.com/BishopFox/sliver) | `docker/offsec-agents/Dockerfile.sliver-tools` |
| Loki | [boku7/Loki](https://github.com/boku7/Loki) | `docker/offsec-agents/Dockerfile.loki-tools` |
| AdaptixC2 | [Adaptix-Framework/AdaptixC2](https://github.com/Adaptix-Framework/AdaptixC2) | `docker/offsec-agents/Dockerfile.adaptix-tools` |

### Deployment Pattern
- Container runs continuously with dormant activation (monitors for activation flag file)
- Activation/deactivation via API endpoint per operation
- One deployment per operation at a time
- Each follows `docker/offsec-agents/burp-entrypoint.sh` dormant pattern
- Reiterate deployment until error-free

**Success criteria:** Each Dockerfile builds successfully. Containers start in dormant mode. Activation/deactivation works per operation.
