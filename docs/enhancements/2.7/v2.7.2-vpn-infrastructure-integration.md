**Functionality:**
```bash
#!/bin/bash

# Monitor /etc/openvpn/configs/*.ovpn for new configs
# Monitor /etc/wireguard/configs/*.conf for new configs
# When config added, validate and attempt connection
# Maintain connection with auto-reconnect
# Log all connection events
# Expose metrics endpoint (connection status, IP, routing table)

case "$1" in
  daemon)
    echo "Starting VPN Manager Daemon..."
    # Run connection monitor loop
    while true; do
      check_openvpn_configs
      check_wireguard_configs
      maintain_connections
      update_routing
      sleep 5
    done
    ;;
  connect)
    # Manually connect to specific config
    connect_vpn "$2" "$3"
    ;;
  disconnect)
    # Disconnect from specific tunnel
    disconnect_vpn "$2"
    ;;
  status)
    # Print current VPN status
    show_vpn_status
    ;;
esac
```

CREATE TYPE vpn_type AS ENUM ('openvpn', 'wireguard');
CREATE TYPE vpn_status AS ENUM ('disconnected', 'connecting', 'connected', 'error', 'reconnecting');

CREATE TABLE vpn_configs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  
  -- Config metadata
  name TEXT NOT NULL,
  description TEXT,
  vpn_type vpn_type NOT NULL,
  
  -- File info
  filename TEXT NOT NULL,
  config_content TEXT NOT NULL, -- Stored encrypted
  file_hash TEXT NOT NULL,
  
  -- Extracted metadata
  remote_host TEXT,

  protocol TEXT,

  -- Association
  operation_id UUID REFERENCES operations(id) ON DELETE SET NULL,

  status vpn_status DEFAULT 'disconnected',
  is_active BOOLEAN DEFAULT true,
  
  -- Connection tracking
  connected_at TIMESTAMP,
  disconnected_at TIMESTAMP,
  last_connection_attempt TIMESTAMP,
  connection_failures INTEGER DEFAULT 0,
  
  -- Assigned IP from VPN server
  assigned_ip TEXT,
  assigned_subnet TEXT,
  gateway_ip TEXT,
  dns_servers TEXT[],
  
  -- Auto-reconnect
  auto_reconnect BOOLEAN DEFAULT true,
  max_reconnect_attempts INTEGER DEFAULT 5,
  
  -- Security
  is_encrypted BOOLEAN DEFAULT true,
  encryption_key_id TEXT,
  
  -- Routing
  route_all_traffic BOOLEAN DEFAULT false,
  custom_routes TEXT[], -- CIDR notation
  
  -- Metadata
  metadata JSON DEFAULT '{}',
  
  -- Audit
  uploaded_by UUID NOT NULL REFERENCES users(id),
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE vpn_connection_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  vpn_config_id UUID NOT NULL REFERENCES vpn_configs(id) ON DELETE CASCADE,
  
  -- Event details
  event_type TEXT NOT NULL, -- 'connected' | 'disconnected' | 'error' | 'reconnect_attempt'
  message TEXT,
  error_details TEXT,
  
  -- Connection metrics
  assigned_ip TEXT,
  bytes_sent BIGINT,
  bytes_received BIGINT,
  connection_duration_seconds INTEGER,
  
  -- Metadata

);

import { createCipheriv, createDecipheriv, randomBytes } from "crypto";

async function encryptVPNConfig(content: string): Promise<{
  encrypted: string;
  keyId: string;
}> {
  const key = randomBytes(32); // AES-256
  const iv = randomBytes(16);
  const cipher = createCipheriv("aes-256-cbc", key, iv);
  
  let encrypted = cipher.update(content, "utf8", "base64");
  encrypted += cipher.final("base64");
  
  // Store key in secure keystore (env var or vault)
  const keyId = await storeEncryptionKey(key, iv);
  
  return { encrypted, keyId };
}

async function decryptVPNConfig(encrypted: string, keyId: string): Promise<string> {
  const { key, iv } = await retrieveEncryptionKey(keyId);
  const decipher = createDecipheriv("aes-256-cbc", key, iv);
  
  let decrypted = decipher.update(encrypted, "base64", "utf8");
  decrypted += decipher.final("utf8");
  
  return decrypted;
}

## Connection Workflow

### OpenVPN Connection Sequence
```
1. User uploads .ovpn file → Frontend
2. Validate config format → Backend
3. Extract metadata (remote, port, auth) → Backend
4. Encrypt and store config → Database
5. Copy config to VPN manager container → Docker
6. User clicks "Connect" → Frontend
7. Execute: openvpn --config /etc/openvpn/configs/client.ovpn → Container
8. Monitor connection status → Service
9. Update status in DB: connecting → connected → Database
10. Configure iptables NAT rules → Container
11. Update container routing (if enabled) → Docker network
12. Display status in UI → Frontend
```

### WireGuard Connection Sequence
```
1. User uploads .conf file → Frontend
2. Validate config format → Backend
3. Extract interface and peer config → Backend
4. Encrypt and store config → Database
5. Copy config to /etc/wireguard/ → Container
6. User clicks "Connect" → Frontend
7. Execute: wg-quick up wg0 → Container
8. Monitor connection with: wg show → Service
9. Update status: connected → Database
10. Configure routing rules → Container
11. Update container routing → Docker network
12. Display status in UI → Frontend
```

## Routing Configuration

### Container-to-VPN Routing Methods

**Method 1: Network Namespace Sharing** (Simplest)
```yaml
# Make tools container share VPN network namespace
rtpi-tools:
  network_mode: "container:rtpi-vpn-manager"
```

**Pros:** All traffic automatically routes through VPN  
**Cons:** Cannot disable routing without restart

**Method 2: Custom Routes via iptables** (Recommended)
```bash
# In VPN manager container
# NAT traffic from specific containers
iptables -t nat -A POSTROUTING -s 172.20.0.0/16 -o tun0 -j MASQUERADE
iptables -A FORWARD -s 172.20.0.0/16 -o tun0 -j ACCEPT
iptables -A FORWARD -i tun0 -d 172.20.0.0/16 -m state --state RELATED,ESTABLISHED -j ACCEPT
```

**Pros:** Granular control, can enable/disable per container  
**Cons:** Requires manual IP management

**Method 3: Docker User-Defined Network** (Most Flexible)
```yaml
networks:
  rtpi-vpn-routed:
    driver: bridge
    driver_opts:
      com.docker.network.bridge.name: br-vpn
    ipam:
      config:
        - subnet: 172.30.0.0/16
          gateway: 172.30.0.1 # VPN manager container
```

**Pros:** Native Docker, clean architecture  
**Cons:** Requires network design planning
