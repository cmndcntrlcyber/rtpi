## v2.8.1 — LLM Chatbot Enhancement

### Phase 1: Operations Manager LLM Chat (COMPLETE)

Persistent conversation system with database-backed message history, replacing the previous in-memory-only chat.

#### What Was Implemented

| Component | File | Description |
|-----------|------|-------------|
| Database schema | `shared/schema.ts` | `agentConversations` and `agentConversationMessages` tables with enums |
| Migration | `migrations/0031_add_agent_conversations.sql` | SQL migration with indexes for efficient querying |
| Conversation service | `server/services/agent-conversation-service.ts` | CRUD operations for conversations and messages |
| Agent chat API | `server/api/v1/agent-chat.ts` | REST endpoints with operation context building |
| Route registration | `server/index.ts` | Mounted at `/api/v1/agent-chat` |
| Frontend hook | `client/src/hooks/useAgentChat.ts` | `useAgentChat()` hook with auto-load, send, clear |
| Chat UI upgrade | `client/src/components/operations/OpsManagerFloatingChat.tsx` | Persistent conversations, provider display, clear button |

#### New API Endpoints

| Method | Endpoint | Purpose |
|--------|----------|---------|
| POST | `/api/v1/agent-chat/:agentRole/message` | Send message, get AI response (persisted) |
| GET | `/api/v1/agent-chat/:agentRole/conversation` | Load active conversation + messages |
| GET | `/api/v1/agent-chat/conversations` | List all conversations for user |
| GET | `/api/v1/agent-chat/:conversationId/messages` | Get messages for specific conversation |
| DELETE | `/api/v1/agent-chat/:agentRole/conversation` | Archive conversation (soft delete) |

#### Database Tables Added

- `agent_conversations` — Conversation sessions with metadata (title, message count, token usage, provider info)
- `agent_conversation_messages` — Individual messages with AI metadata (provider, model, tokens, duration)

#### Key Features

- Conversations persist across browser sessions and page reloads
- Automatic conversation title generation from first user message
- Token usage tracking per message and per conversation
- AI provider/model display on assistant messages
- "New conversation" button to archive and start fresh
- Conversation history loaded from DB (up to 40 messages for context)
- Operation context automatically built and injected into system prompt
- Extensible agent role system (ready for Phase 2 individual agent chats)
